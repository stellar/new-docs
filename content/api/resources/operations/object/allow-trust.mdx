Updates the “authorized” flag of an existing trust line. This must be called by the issuer of the asset.

See the [`Allow Trust` errors](../../../errors/result-codes/operation-specific/allow-trust.mdx).

<AttributeTable>

- ATTRIBUTE
  - DATA TYPE
  - DESCRIPTION
- asset_type
  - string
  - The type of asset. Either `native`, `credit_alphanum4`, or `credit_alphanum12`.
- asset_code
  - string
  - The Stellar address of the asset.
- asset_issuer
  - string
  - The code for the asset.
- authorize
  - int
  - Flag indicating whether the trustline is authorized.  1 if the account is authorized to transact with the asset. 2 if the account is authorized to maintain orders, but not to perform other transactions. 0 if the account is not authorized to transact with the asset in any way.
- trustee
  - string
  - The issuing account, or source account in this instance.
- trustor
  - string
  - The trusting account, or the account being authorized or unauthorized.

</AttributeTable>
